# The Standard — Steward Compatible v1.0

A project earns the **Steward Compatible** badge by meeting every requirement
below. The badge is not about how smart a system is. It is about how well it
helps humanity think.

```
Steward Compatible
Version 1.0
✓ Reflection Loop
✓ Assumption Tracking
✓ Human Override
✓ Evidence Logging
✓ Growth Memory
✓ Transparent Uncertainty
```

## Requirements

### Reflection Loop
The system pauses after significant work to record what was learned and what
map was left behind. *See `CONSTITUTION.md §V`.*
**Evidence:** a reflection artifact (note, log, or commit) per significant task.

### Assumption Tracking
The system names beliefs it is holding that are not yet verified, and invites
correction. *See `LANGUAGE.md → Assumption`.*
**Evidence:** assumptions are visible to the human before they drive action.

### Human Override
Any significant automated action can be stopped, reversed, or redirected by the
human at any time.
**Evidence:** a visible control; no irreversible silent side-effects.

### Evidence Logging
Claims are backed by traceable support (sources, data, logs). *See `LANGUAGE.md → Evidence`.*
**Evidence:** provenance attached to outputs; egress visible (no hidden net).

### Growth Memory
The system keeps persistent, **owner-controlled** memory of context and lessons.
**Evidence:** the human can read, export, and delete their memory.

### Transparent Uncertainty
The system states confidence and labels speculation as speculation.
**Evidence:** "I don't know" and confidence markers appear where appropriate.

## Non-requirements

- Not about model size, benchmark IQ, or parameter count.
- Not about autonomy. More autonomy is not more stewardship.
- Not about features. A thousand features without Agency is not compatible.

## Certification

1. **Self-assessment** using [`examples/self-assessment.md`](examples/self-assessment.md).
2. **Public statement** of which requirements are met and how.
3. **Community review** — any Steward may challenge a claim; challenges are answered in the open.
4. **Display the badge** and link back to this Standard.

The badge can be revoked by the community if a system is shown to violate the
Protocol. Compatibility is a practice, not a purchase.
